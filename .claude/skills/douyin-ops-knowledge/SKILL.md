# 抖音电商运营知识库

## 触发条件

当用户需要：
- 分析抖店数据，诊断运营问题
- 了解抖音算法机制、流量分发规则
- 优化商品卡转化率、降低退款率
- 制定投放策略、达人合作策略
- 旺季备战、大促活动策划
- 问"为什么数据差了""该怎么办"

## 核心能力

本 skill 提供结构化运营知识，覆盖六大模块：

| 模块 | 文件 | 内容 |
|------|------|------|
| 算法机制 | `references/algorithm.md` | 流量分发逻辑、体验分体系、商品卡入池、搜索排名 |
| 平台规则 | `references/rules.md` | 新手期考核、违规处罚、评分体系、仅退款规则 |
| 实战策略 | `references/strategies.md` | 起店打法、商品卡优化、千川投放、达人合作、内容策略 |
| 运营SOP | `references/sop.md` | 每日/每周/每月运营动作清单 |
| 诊断手册 | `examples/diagnosis-playbook.md` | 常见问题诊断树：流量问题/转化问题/退款问题 |
| 旺季方案 | `examples/peak-season.md` | 618/暑假/开学季/双11 运营策略 |

## 使用方式

1. **分析数据时**：对照 `examples/diagnosis-playbook.md` 的诊断树，逐层排查
2. **制定策略时**：参考 `references/strategies.md` 选择适合当前阶段的打法
3. **日常运营时**：按 `references/sop.md` 执行，不遗漏
4. **旺季备战时**：按 `examples/peak-season.md` 的时间线推进

## 核心原则

1. **数据驱动**：所有判断基于店铺真实数据，不凭感觉
2. **循序渐进**：先解决最大问题（如退款率），再优化增长（如转化率）
3. **安全第一**：所有操作遵守平台规则，不刷单不违规
4. **内容为王**：抖音是内容平台，商品卡+短视频+直播三场协同
