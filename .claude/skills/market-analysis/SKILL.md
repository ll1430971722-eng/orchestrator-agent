# 市场分析 — 统一入口

## 概述

这是市场分析的**唯一入口**。一个产品/想法进来，三阶段接力分析，产出一份完整报告。

```
用户输入：/market-analysis <产品/想法描述> --depth [quick|standard|deep]

Stage 1: 创意验证 → 这个想法值不值得做？
Stage 2: 用户洞察 → 谁在买、为什么买？
Stage 3: 竞品分析 → 谁在做、我们的机会在哪？
Stage 4: 汇总报告 → 一份完整报告输出到 output/reports/
```

**和旧版的关键区别**：三个阶段信息自动传递，你不需要重复输入。Stage 2 带着 Stage 1 的结论去定位用户，Stage 3 带着 Stage 2 的用户洞察去找竞品空白。

---

## 启动：收集输入

问用户以下核心信息，如果用户已经在指令里提供了，直接进入 Stage 1：

```
1. 产品/想法是什么？（一句话说清楚）
2. 目标客户是谁？
3. 解决什么痛点？
4. 分析深度？
   - quick：速览，5-8min，给方向感
   - standard：标准，15-20min，完整分析（默认）
   - deep：深度，30-40min，含多渠道数据交叉验证
```

**深度分档对每个阶段的影响：**

| 阶段 | quick | standard | deep |
|------|-------|----------|------|
| Stage 1 创意验证 | 只做核心 3 问 | 完整 6 步 | 完整 6 步 + 2 轮自我辩论 |
| Stage 2 用户洞察 | 1 个核心用户群 | 3 类用户画像 | 4 类画像 + 多渠道口碑交叉验证 |
| Stage 3 竞品分析 | 2 个竞品 | 3-5 个竞品 | 5-7 个竞品 + 招聘/融资信号 |
| Stage 4 汇总报告 | 一页纸摘要 | 完整报告 | 完整报告 + 行动路线图 |

## Stage 1：创意验证

### 目标

判断这个想法值不值得继续往下分析。如果是个明显不行的想法，在这里就停下来，不浪费后面时间。

### 流程

**1.1 创意简报**（补全用户没提供的信息）
- 一句话描述
- 目标客户（具体是什么人，不是"所有人"）
- 解决的痛点（客户现在怎么凑合的？）
- 客户愿意为这个付多少钱？

**1.2 市场存在性验证**（并行搜索）
同时搜三个方向，确认确实有人需要这个东西：
- `"[痛点描述] 怎么办"` — 看求助帖
- `"[产品品类] 推荐"` — 看需求讨论
- `"[产品品类] 吐槽"` — 看现状不满

**1.3 自我辩论**
站在反方，用真实质疑挑战这个想法：
- 为什么现在没有人在做？（如果市场真有需求的话）
- 客户现状方案"够用"，为什么要换？
- 最大的单点失败风险是什么？

**1.4 Go/No-Go 判断**

给出明确判断：
- ✅ **Go**：市场有需求、痛点真实、差异化空间存在 → 进入 Stage 2
- ⚠️ **Go with caution**：有机会但风险大，标注关键风险 → 进入 Stage 2
- ❌ **No-Go**：市场不存在、伪需求、或已有不可逾越的壁垒 → 结束

**deep 模式附加**：再做一轮深化辩论，专门攻击第一轮最弱的论点。

### 输出格式

```markdown
## Stage 1：创意验证结论

### 创意简报
[补全后的简报]

### 市场信号
- 需求讨论：[N] 条相关讨论
- 吐槽信号：[N] 条不满
- 付费意愿：[有/无/不确定]

### 核心风险
1. [风险1]
2. [风险2]

### 判断：✅ Go / ⚠️ Go with caution / ❌ No-Go
[理由]
```

**停下来。**
```
Stage 1 完成。判断：[Go / No-Go]
输入"继续"进入 Stage 2，或提出质疑。
```

## Stage 2：用户洞察

### 前置信息（从 Stage 1 自动带入）

- 产品描述
- 目标客户
- 核心痛点
- Stage 1 识别的主要风险

### 目标

理解谁会买、为什么买、什么情绪驱动他们掏钱。不做泛泛的"25-35岁女性"这种画像，要做有画面感的真实行为画像。

### 流程

**2.1 用户画像生成**

用以下框架生成用户画像（数量根据深度模式决定）：

```
要求：
1. 从"用户真实行为"角度切入，而非年龄性别等常规标签
2. 每类画像包含：
   - 用户类型名称（具象化命名）
   - 真实行为描述（具体场景/动作）
   - 消费频次/场景
   - 价值判断（复购潜力、抗周期性、成瘾性）
3. 明确标注核心用户群，并给出理由
4. 让读者能"看到画面感"
```

**2.2 情绪动因分析**（对核心用户群做三层拆解）

- **痛点**：用户最怕什么？什么让他们觉得"被背叛"？
- **痒点**：用户在通过产品表达什么身份？
- **爽点**：那个"啊～"的满足瞬间是什么？

**deep 模式附加**：多渠道交叉验证
- 小红书搜：`"[产品] 种草"` + `"[产品] 后悔"`
- 电商评论区：看真实买家的好评/差评关键词
- 知乎搜：`"[品类] 怎么选"`

**2.3 产品机会清单**

把情绪洞察转化为可落地的产品方向，按优先级排：
- **P0**（核心）：明天就能干，直接打核心痛点
- **P1**（重要）：1-2 周内可落地
- **P2**（锦上添花）：中长期规划

### 输出格式

```markdown
## Stage 2：用户洞察

### 核心用户：⭐ [用户群名称]
- 行为特征：[具体场景]
- 核心情绪：[痛点/痒点/爽点]
- 一句话抓住他：[可用于广告语]

### 情绪动因
- 痛点：[他们真正怕的是什么]
- 痒点：[他们想成为谁]
- 爽点：[什么让他们满足]

### 产品机会
- P0：[最重要的 1-3 个方向]
- P1：[次要方向]
- P2：[锦上添花]
```

**停下来。**
```
Stage 2 完成。核心用户：[xxx]，核心机会：[P0 方向]
输入"继续"进入 Stage 3，或深入某个用户群。
```

## Stage 3：竞品分析

### 前置信息（从 Stage 1 + 2 自动带入）

- 产品描述和定位
- 核心用户画像
- 用户核心痛点
- 我们的 P0 产品机会方向

### 目标

搞清楚谁已经在服务这些用户了、他们做得怎么样、我们的空白机会在哪。

### 流程

**3.1 竞品发现**（并行搜索）

同时搜四个方向找竞品：
- `"best [产品品类] 2025 2026"`
- `"[品类] 品牌 排行"`
- `"[用户痛点] 解决方案"`
- `"[品类] 替代品"`

给出 N 个竞品清单（数量依据深度模式），每个一句话描述。让用户确认或替换。

**停下来。**
```
找到 [N] 个竞品：[列出名单]
输入"确认"继续深入分析，或调整名单。
```

**3.2 逐家深度研究**（并行搜索，所有竞品同时跑）

对每个竞品，同时搜四个维度：
- **产品维度**：功能、技术栈、近期更新
- **定价维度**：价格档位、试用策略、用户对定价的吐槽
- **市场维度**：目标客户、核心营销信息、融资/团队规模
- **口碑维度**：搜 `"[竞品名] 评价"` + `"[竞品名] 吐槽"` + `"[竞品名] 优缺点"`

记录反复出现的吐槽（出现 3 次以上 = 真问题）和反复出现的赞誉。

**3.3 差距矩阵**

| 功能维度 | 我们 | [竞品1] | [竞品2] | [竞品3] |
|---------|------|---------|---------|---------|
| [维度1] | - | 有 | 有 | 无 |
| [维度2] | - | 无 | 有 | 有 |
| [维度3] | - | 无 | 无 | 无 |

对每个功能分类：
- **基础标配**：所有人都有（必须做）
- **差异化**：只有 1-2 家有
- **空白机会**：没人做（我们的机会⭐）

**3.4 定位建议**

基于前面所有分析，给出：
- 我们比谁都好的**那一件事**（单点切入）
- 先抢哪个竞品的不满客户（为什么）
- 传播角度（用竞品最大槽点做文章）

### 输出格式

```markdown
## Stage 3：竞品分析

### 竞品格局
[一句话总结竞争格局]

### 差距矩阵
[表格]

### TOP 3 空白机会
1. [机会1] — 为什么没人做 / 我们能不能做
2. [机会2]
3. [机会3]

### 推荐定位
对于 [目标客户]，他们最大的痛点是 [xxx]。
[产品名] 能带来 [核心价值]。
和 [主要竞品] 不同，我们 [核心差异]。
```

**停下来。**
```
Stage 3 完成。确定了 [N] 个空白机会，核心定位：[xxx]
输入"汇总"生成完整报告。
```

## Stage 4：汇总报告

### 目标

把三阶段产出合成一份完整报告，输出到文件。

### 报告结构

```markdown
# 市场分析报告：[产品/想法名称]
> 分析日期：YYYY-MM-DD | 深度：[quick/standard/deep]

---

## 一句话总结
[这个产品/想法值不值得做，用一句话说清楚]

---

## 1. 创意评估
[Stage 1 摘要：判断 + 核心风险]

## 2. 用户洞察
[Stage 2 摘要：核心用户 + 核心情绪 + P0机会]

## 3. 竞品分析
[Stage 3 摘要：竞品格局 + TOP空白机会 + 推荐定位]

---

## 行动建议（按优先级）

### 明天就做
1. [P0 行动项]
2. [P0 行动项]

### 本周
1. [P1 行动项]

### 本月
1. [P2 行动项]

---

## 待验证假设
- [ ] [假设1] — 验证方式：[xxx]
- [ ] [假设2] — 验证方式：[xxx]

---

> 数据来源：[列出所有引用的搜索和来源]
> 分析方法：market-analysis pipeline v1.0
```

### 输出

报告保存到 `output/reports/market-analysis-<产品名>-YYYY-MM-DD.md`

## 搜索规则

- Stage 2 和 Stage 3 的关键搜索并行执行（同时搜多个方向）
- 用户口碑优先看真实评价平台（小红书、知乎、电商评论区）
- 招聘 JD 能暴露竞品技术栈和团队方向
- 融资/工商信息能反映市场走向和竞品实力
- **所有发现标注来源**，不确定的信息标"待确认"

## 安全规则

- 只读搜索，不做任何修改性操作
- 不购买、不注册、不留下痕迹
- 不抓取付费墙后的内容
- 报告中不出现 API Key、Token、密码
