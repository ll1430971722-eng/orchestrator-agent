## 触发条件

当需要执行完整日常分析流程时使用。适合每天运营结束后做全面分析。

## 功能

编排完整流程：回头看 → 数据采集 → 指标计算 → 问题诊断 → 行动清单 → 未来建议 → 日报生成。

## 前置条件

- Playwright MCP 可用（数据采集阶段需要）
- 数据文件在 `data/raw/` 下（如已有今日数据则跳过采集）
- Python 3 可用（数据分析阶段需要）

## 完整流程

---

### Phase 1: 检查并准备数据

1. 检查 `data/raw/orders/YYYY-MM-DD/` 是否存在今日数据
2. 如果数据已存在 → 询问用户："今日数据已存在，是否重新拉取？"
3. 如果数据不存在 → 自动调用 `douyin-fetch-data` 技能拉取数据
4. 如果 Playwright MCP 不可用 → 提示用户手动运行或提供数据

### Phase 2: 回头看上次建议

**目的：** 检查上次分析给出的建议执行了没有、效果怎么样。这是整个工作流的"反馈闭环"——没有这一步，分析就是在自说自话。

**步骤：**

1. **找上一次的日报和行动清单**
   - 读取 `output/reports/` 下最近一次（非今天）的 `*-daily-report.md` 和 `*-action-plan.md`
   - 如果找不到（首次运行），跳过本环节，在日报中标注"首次分析，无历史建议可回顾"

2. **逐条检查执行状态**
   - 列出上次的所有行动建议
   - 对于每条建议，判断执行状态：
     - ✅ 已执行（数据有对应变化，或用户已确认完成）
     - 🔄 进行中（部分完成，指标有变化但未达预期）
     - ❌ 未执行（数据无变化，需询问原因）
   - 对于不确定状态的，在报告中标注"待确认"，请用户补充

3. **对比指标看效果**（只对"已执行"和"进行中"的）
   - 看建议对应的指标有没有朝预期方向变化
   - 例：上次建议"优化详情页降退款" → 对比上次和今天的退款率
   - 标注效果：✅ 有效 / ⚠️ 未见效果 / 🔄 观察中（数据不够，需更长时间）

4. **把回头看结论写入日报**
   - 在日报开头（一句话总结之前）加"📮 上次建议回头看"章节

5. **指导本轮建议**
   - 上次有效的动作 → 本轮继续推荐或扩大范围
   - 上次无效的动作 → 本轮换方向，不重复无效建议
   - 上次未执行的动作 → 本轮询问原因，判断是否仍需做，如有障碍帮解决

### Phase 3: 计算核心指标

运行分析脚本：

```bash
python scripts/analyze_daily.py
```

脚本会读取 `data/raw/` 下的数据，计算：
- GMV（成交总额）
- 订单数
- 退款金额 & 退款率
- 客单价
- 商品销量排行
- 退款原因排行

输出到 `data/processed/daily_metrics_YYYY-MM-DD.json`

**如果脚本报错**（数据格式不匹配等）：
- 读取 raw 数据文件
- 直接在上下文中计算指标（GMV = sum(pay_amount)，退款率 = refund_count / order_count 等）
- 不阻塞流程

### Phase 4: 问题诊断

运行诊断脚本：

```bash
python scripts/diagnose_problems.py
```

或直接在上下文中基于指标做诊断：

**诊断规则**（参考 `docs/metrics_rules.md`）：
- 退款率 > 5% → 高优先级告警
- GMV 环比下降 > 20% → 高优先级
- 单品销量集中度 > 80% → 中优先级（单一爆款风险）
- 广告 ROI < 1.5 → 建议优化投放
- 流量环比下降 > 30% → 高优先级

**每个问题包含**：
- 数据依据（具体数字）
- 可能原因（按可能性排序）
- 验证方法
- 解决方案
- 新手解释

### Phase 5: 生成行动清单

基于诊断结果，生成按优先级排序的行动清单：

| 优先级 | 示例 |
|--------|------|
| 🔴 今天必须做 | 检查高退款率商品的具体退货原因 |
| 🟡 本周内做 | 优化低转化率商品的主图和详情 |
| 🟢 持续观察 | 新上架商品的数据表现（需要 1-2 周观察期）|

### Phase 6: 生成未来建议

- **未来 3 天**: 立即能做的优化动作（如调整广告出价、优化商品标题）
- **未来 7 天**: 观察重点（验证优化效果、观察趋势变化）
- **未来 30 天**: 战略方向（品类拓展、内容策略调整）

### Phase 7: 生成统一日报

保存到 `output/reports/daily_report_YYYY-MM-DD.md`

## 输出物

| 文件 | 路径 |
|------|------|
| 处理后的指标 | `data/processed/daily_metrics_YYYY-MM-DD.json` |
| 日报 | `output/reports/daily_report_YYYY-MM-DD.md` |
| 行动清单 | `output/reports/action_plan_YYYY-MM-DD.md` |
| 未来建议 | `output/reports/future_recommendations_YYYY-MM-DD.md` |

## 安全规则

- ✅ 只读取本地数据文件
- ✅ 所有建议均为人工执行建议
- ❌ 不自动执行任何店铺修改
- 报告中标注"Agent 不自动执行修改"
