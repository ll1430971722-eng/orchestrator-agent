# 竞品分析

## 概述

通过网页搜索对 3-5 个竞争对手做深度研究。产出一份功能对比矩阵、用户口碑摘要、和清晰的差异化定位策略。可独立使用，也可作为 `/idea-validator` 的后续步骤。

## 第一步：确定竞品名单

问用户：「你最关心的 3-5 个竞品是谁？如果不确定，我来帮你找。」

如果需要我来找，搜索以下关键词：
- `"best [产品品类] 2025"`
- `"[细分领域] 品牌 排行"`
- `"[细分领域] 替代品"`
- `"[痛点] vs"` — 对比类文章里往往藏着竞品
- `"[细分领域] 融资"` — Crunchbase/36氪

给出一份 3-5 个竞品的简短清单（每个一句话描述），让用户确认或替换。

**停下来。**
```
第一步完成。输入"确认"继续深入分析每个竞品，或者调整名单。
```

---

## 第二步：逐家深度研究

对每个竞品，通过网页搜索收集以下信息：

**产品情报：**
- 功能列表（看官网、更新日志、帮助文档）
- 技术栈（招聘 JD 里常有线索）
- 集成和生态
- 近期产品更新

**定价情报：**
- 所有价格档位及包含内容
- 免费版/试用策略
- 企业版定价（即使未公开也尽量找）
- 用户对定价的吐槽

**市场位置：**
- 目标客户群（ICP）
- 核心营销信息
- 融资阶段和金额
- 团队规模（LinkedIn 或企查查）

**用户口碑：**
对每个竞品搜索：
- `"[竞品名] 评价"`
- `"[竞品名] 吐槽"`
- `"[竞品名] 优缺点"`
- `"[竞品名] 太贵"`
- `"[竞品名] 替代品"` — 用户为什么要换掉它？

记录反复出现的吐槽（出现 3 次以上 = 真问题）和反复出现的赞誉。

以结构化摘要的形式呈现每个竞品的研究结果。

**停下来。**
```
第二步完成。输入"确认"进入差距分析，或对某个竞品补充意见。
```

---

## 第三步：功能矩阵 + 差距分析

输出对比矩阵：

| 功能维度 | 我们 | [竞品1] | [竞品2] | [竞品3] |
|---------|------|---------|---------|---------|
| [功能] | - | 有 | 有 | 无 |
| [功能] | - | 无 | 有 | 有 |
| [功能] | - | 无 | 无 | 无 |

用 `-` 表示我们（待填充），用 `有` / `无` / `部分` 表示竞品。

**对每个功能分类：**
- 基础标配 — 所有人都有（必须做）
- 差异化 — 只有 1-2 家有
- 空白机会 — 没人做（我们的机会）

明确标出 TOP 3 空白机会。

**停下来。**
```
第三步完成。输入"确认"进入定位策略，或修改矩阵内容。
```

---

## 第四步：定位策略

基于差距分析和口碑研究：

**推荐定位：**
- 我们做得比谁都好的那一件事（切入点）
- 现有方案服务不到的客户群
- 先抢哪个竞品的不满客户（为什么）
- 抢这些客户的传播角度（拿他们吐槽竞品的第一大槽点做文章）

**定位声明模板：**
> 对于 [目标客户] 来说，他们最大的痛点是 [核心痛点]，[产品名] 是一个 [品类]，能带来 [核心价值]。和 [主要竞品] 不同，我们 [核心差异]。

**停下来。**
```
第四步完成。竞品分析全部结束。
输入"确认"接受这一定位，或提出修改方向。
```

---

## 搜索规则

- 每个竞品至少 4-5 次搜索
- 用户口碑优先看真实评价平台（小红书、知乎、电商评论区），这些比官网诚实
- 招聘 JD 能暴露技术栈和团队方向
- 融资/工商信息显示市场走向和竞品实力
