# 1688 & 抖音运营速查词表

> 厂二代扫盲专用。看不懂数据的时候，回来查这个。

## 一、1688 核心指标

### 店铺L等级
1688 对店铺的综合评级，L0-L6。**等级越高，拿到的免费流量越多**。
- 核心权重：GMV（成交额）、买家数、转化率、BSR评分
- 新手目标：L2-L3 算及格，L4 算不错
- **如果L等级不动 → 说明核心指标没有突破**

### BSR评分（Business Service Rating）
买家服务评分，1-5分。**低于4.0会降权**。
- 货描相符：产品质量满意度
- 响应速度：旺旺回复时间
- 发货速度：48小时发货率
- 回头率：老客户复购
- **查看位置**：卖家工作台 → 店铺管理 → BSR评分

### 自然流量
不花钱的流量，来自搜索排名和平台推荐。
- **影响因素**：店铺L等级、BSR评分、商品标题/主图/详情页质量、买家好评率
- **如果自然流量下降 → 要么竞争对手在抢，要么你店铺数据变差了**

### 金冠商品
1688 的"优质商品"标签，搜索加权。
- 通过销量、转化率、买家评价等自动评出
- **目标是店铺里越多金冠商品越好**

### 迅捷回复（旺旺响应）
买家有购买意向的信号。
- **如果咨询多但成交少 → 可能是回复速度慢、价格问题、或客服话术问题**
- 目标：**首次响应 < 30秒，平均响应 < 60秒**

### 客单价
平均每单多少钱。
- 低客单价 = 走量，高客单价 = 利润
- 如果**访客多但客单价低** → 要么引流的是低价值客户，要么定价/搭配有问题

## 二、抖音小店 核心指标

### 体验分（DSR）
抖音对店铺的综合评分，**影响所有流量入口**。
- **体验分 ≥ 4.5**：正常获取流量
- **体验分 < 4.0**：限制流量
- 包含三个子项：商品体验（差评率）、物流体验（发货速度）、服务体验（客服响应+退货率）
- **查看位置**：抖店后台 → 店铺 → 体验分

### 商品卡流量
**抖音最稳定的免费流量来源**。
- 用户通过搜索商品名称找到你的商品页
- **优化方向**：商品标题包含高频搜索词、主图清晰有卖点、价格有竞争力
- **如果商品卡流量高但转化低 → 价格或详情页有问题**

### 精选联盟
达人选货平台。达人通过精选联盟看到你的商品，帮你带货。
- 佣金比例通常 10%-30%
- **核心动作**：设置合理的佣金 → 达人看到会主动带
- 如果达人带货多但销量一般 → 选错达人了（粉丝画像不匹配）

### 千川（巨量千川）
抖音的广告投放系统。
- **关键指标**：
  - ROI（投产比）：销售额 ÷ 广告花费。> 1 才赚钱
  - CPM（千次展示成本）：越低越划算
  - 点击率：广告文案和素材的吸引力
- **新手建议**：先ROI跑正，再放量。不要一上来就猛花钱

### 猜你喜欢（推荐流量）
抖音商城首页的推荐位流量。
- 主要取决于：**商品点击率、转化率、体验分**
- 如果这个流量占比高 → 说明商品"体质"好，平台愿意推

### 客单价
同1688。但抖音的客单价一般比1688高（零售价 vs 批发价）。

## 三、共同概念

### 转化率（最重要的指标）
访客中最终成交的比例。
- **1688**: 行业平均1-3%，好的能到5%+
- **抖音**: 商品卡转化率 2-5%，千川1-3%
- 转化率低 → 通常是价格、详情页、或者流量不精准的问题

### 同比 vs 环比
- **同比**：和去年同一天比 —— 看长期趋势
- **环比**：和昨天比 —— 看短期波动
- 运营日报里两种都要看

### 异常信号清单
| 信号 | 可能原因 |
|------|----------|
| 访客涨但转化跌 | 流量不精准（比如投流人群跑偏） |
| 访客突然暴跌 | 被限流/被降权 |
| 转化率突然涨 | 可能是有大单或特殊活动 |
| 广告花费翻倍但GMV不涨 | ROI 出问题了 |
| 体验分/BSR突然掉 | 差评爆发或发货延迟 |

## 四、每天看日报时的提问套路

问运营这几个问题，对方就知道你不是外行：

1. **今天的关键数字是多少，你觉得正常吗？**
2. **如果不好，你觉得主要原因是什么？**
3. **你打算怎么改？**
4. **如果好，是因为什么做对了？能不能复制？**

进阶版（能镇住场）：
- **"今天访客涨了但转化率掉了，是新引流渠道的人群不精准，还是价格调整的问题？"**
- **"你这个广点通/千川的ROI是多少？跑正没有？"**
- **"最近BSR/体验分有什么变化吗？有没有差评要处理的？"**

---

> **不要记住所有指标。**
> 前两周的目标只有一个：**看到数据能判断"正常"还是"不对劲"**。
> 不对劲的时候，拿着这个表问运营。
